⏱ Time Complexity
Train: O(n · d · T)
Predict: O(T · depth)
With n=9,600 · d=8 · T=300:
Train ≈ O(23M ops) = high cost
Predict ≈ O(300×5) = 1,500 ops
∑ Core Formula
F₀(x) = argminγ Σ L(yᵢ, γ)
Fₘ(x) = Fₘ₋₁(x) + η·hₘ(x)
hₘ = weak tree fit to −∇Loss
η = 0.08 (learning rate)
Loss = cross-entropy (deviance)

J · Project Conclusion
Summary

Four ML models were evaluated on 12,000 IoT sensor samples from an Arduino weather station using 8 features (5 sensors + 3 time features). After hyperparameter optimisation, Gradient Boosting achieved 99.7% current accuracy — a +10.7% improvement over baseline. Random Forest leads the 2-hour forecast task at 60.1%.

Key Findings
  • Temperature is the most predictive feature (24.1% importance)
  • Gradient Boosting's sequential learning outperforms parallel ensembles for current classification
  • All 4 models exceed 97% accuracy on current weather — IoT data quality is high
  • Forecast accuracy (≤60%) reflects genuine atmospheric unpredictability at 2-hour horizon
  • Live WeatherAPI validation confirms model alignment with real-world conditions
